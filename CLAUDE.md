## Project Overview

`draw-this` is an HTML5 canvas drawing toy. It runs in the browser by opening `index.html`. The application logic is written in TypeScript, bundled by esbuild into `script.js`.

## Development

**Prerequisites:** Node.js with npm.

```bash
npm install           # install dev dependencies (first time only)
npm run build         # compile TypeScript to script.js
npm run watch         # rebuild on file changes
npm run test          # run Vitest in watch mode
npm run test:run      # run tests once
npm run typecheck     # type-check only (no output)
npm run check         # tests + type-check in one command
```

Open `index.html` directly in a browser after building. Changes to `style.css` or `index.html` take effect on refresh. Changes to TypeScript source require a rebuild.

**IMPORTANT: `script.js` is a build artifact generated by esbuild. Never edit it directly. Edit the TypeScript source in `src/` instead.**

## Architecture

### File structure

- **`index.html`** (~145 lines) -- DOM structure organized as `#main > #topbar + #body`. `#topbar` holds `#standard-tools` (oops/nuke/screenshot) and `#top-tools` (tool buttons, filtered by active category). `#body` holds `#sidebar` (with `#colors-col` and `#category-col`) and `#stretcher` (which wraps `<canvas id="drawHere">`). Footer hosts `#stampchooser` and the 7 `<audio>` elements. Button icons are base64-encoded SVGs inlined in `src=` attributes.
- **`style.css`** (~160 lines) -- All styling. Uses flex layout for `#main`/`#topbar`/`#body`/`#sidebar`. Tool visibility is driven by `#top-tools[data-active-category="X"] .tool[data-category="X"] { display: inline-block; }` rules (all `#top-tools .tool` default to `display: none`).
- **`script.js`** -- **BUILD ARTIFACT.** Generated by `npm run build` from `src/main.ts`. Do not edit.
- **`assets/sounds/`** -- 7 MP3 sound effects (click, drip, stamp, eraser, pencil, oops, tornado).

### The tool plugin architecture

The central design decision is that `tools` is a **plugin registry**. Each key is a tool name; each value implements the `Tool` interface from `src/types.ts`. Adding a new tool means adding a new module in `src/tools/`, re-exporting it from `src/tools/index.ts`, and adding a button in `index.html` under `#top-tools` with a `data-category` attribute -- the event wiring in `drawHorse.addListeners()` picks it up automatically.

**Tool interface** (defined in `src/types.ts`):

```typescript
interface Tool {
  name: string
  button: HTMLElement            // reference to toolbar button
  selectable: boolean            // false = one-shot action (oops, nuke)
  drawsImmediately: boolean      // true = draw() fires on click; false = fires on drag
  onclick: (e: Event) => void    // fired when toolbar button clicked
  draw: (e: MouseEvent | TouchEvent) => void    // fired on move-while-held, or on click if drawsImmediately
  stopDrawing: (e: MouseEvent | TouchEvent) => void  // fired on mouseup/touchend
  settings?: ToolSettings        // optional width/size configuration (not yet rendered in UI)
}
```

`drawHorse` is the host that manages all state and event routing. Tools never register their own event listeners -- they only implement the interface above.

### Tool categories (CSS-driven visibility)

Tools in `#top-tools` are grouped by `data-category` (pen, brush, toy, stamp, eraser, filter, bucket, letter). The `#category-col` sidebar column holds one `.category` button per category. Clicking a `.category` button:

1. Sets `#top-tools[data-active-category="<cat>"]`.
2. Adds the `selectedControl` class to the clicked category button, removing it from the previously selected one.

CSS rules `#top-tools[data-active-category="X"] .tool[data-category="X"] { display: inline-block; }` then reveal only the tools in the active category. On page load, `main.ts` marks the `pen` category button as `selectedControl` (the HTML `data-active-category="pen"` attribute provides the initial visibility). Buttons in `#standard-tools` (oops, nuke, screenshot) are always visible because they have no `data-category`.

Stub categories (`filter`, `letter`, and currently `bucket`) have no tools yet; clicking them sets `data-active-category` but `#top-tools` renders empty.

### `drawHorse` object (`src/drawHorse.ts`)

Main application state. DOM references are captured at module-load time (before `window.onload`). Implements `DrawHorseContext` interface from `src/types.ts`.

Key state:
- `ctx` -- 2D canvas context (set in `window.onload`)
- `currentTool` -- reference to the active tool object (set to `tools.pencil` in `main.ts`)
- `selectedColor` -- CSS color name string
- `selectedStamp` -- reference into `stamps` object
- `undoStack` -- array of `ImageData` snapshots; pushed on every mouse-up
- `pos {x, y}` -- current draw position in canvas coordinates
- `stamps` -- 8 active stamps (horse, beetle, anteater, kitty, bunny, manatee, whale, toucan)

Key methods:
- `setPosition(e)` -- converts event coords to canvas-relative using `e.target.getBoundingClientRect()`. **Bug: if `e.target` is not the canvas, offset is wrong.**
- `endPosition(e)` -- called on mouse-up; calls `stopDrawing`, updates pos, pushes undo snapshot
- `resize()` -- sizes canvas to `0.95 * (window.innerHeight - header.offsetHeight - topbar.offsetHeight)`. `#topbar` is looked up lazily via `document.getElementById('topbar')`; if absent, falls back to 0.
- `addListeners()` -- registers all mouse and touch events, plus the click handler for `.category` buttons (drives `#top-tools[data-active-category]` and `selectedControl` state). Event host for the plugin system.
- `setupStamps()` -- dynamically builds `#stampchooser` innerHTML from `stamps` object; assigns click handlers
- `setupColorChooser()` -- assigns click handlers to `.colorChoice` buttons; color is read from `button.id`

### `window.onload` initialization order (in `src/main.ts`)

1. `setupColorChooser()`
2. Assign `drawHorse.ctx`
3. `resize()` (handles all canvas sizing; no manual dimension setup)
4. `addListeners()`
5. Mark `.category[data-category="pen"]` as `selectedControl` (default active category)
6. `setupStamps()`
7. `setupTools()` (no-op, preserved for structural parity)

### Stamp SVG format

Each stamp in `drawHorse.stamps` (defined in `src/stamps.ts`) is `{id, name, url}` where `url` is a base64-encoded SVG. The token `%%%%` in the SVG source is a fill color placeholder.

Rendering: `base64 decode -> replaceAll("%%%%", "style='fill:COLOR;stroke:COLOR;'") -> base64 re-encode -> data URI -> ctx.drawImage`

### Sound (`src/sounds.ts`)

`playSound(id)` / `pauseSound(id)` -- exported functions that operate on `<audio>` elements by DOM id. Looping sounds are started in `tool.draw()` and stopped in `tool.stopDrawing()`.

### Global exports for browser compatibility

`src/main.ts` exposes `tools`, `drawHorse`, `playSound`, and `pauseSound` on `globalThis` so that `index.html` inline scripts and the browser console can access them, preserving the original API surface.

## Known Issues and Incomplete Work

- **`bucket` tool** -- `colorPixel()` hardcodes `[100, 1, 2]` instead of parsing selected color; button hidden in HTML until fixed.
- **Bubble images external** -- 3 SVG URLs still point to `cdn.glitch.global`.
- **Eraser** -- draws white strokes; does not restore transparency; only "erases" against white backgrounds.
- **Undo granularity** -- snapshot on mouse-up means multi-stroke drags between releases share one undo level. `nuke` does not push a snapshot, so it cannot be undone.
- **`setPosition` offset bug** -- tied to `e.target` being the canvas element.
- **`settings` unrendered** -- tools have optional `ToolSettings` with width/size; no UI renders them; `lineWidth` is hardcoded to `5` in each tool's `draw()`.
- **`setupTools()`** -- exists as a method but the body is a no-op loop; preserved for structural parity.

## Adding a New Tool

1. Create `src/tools/your-tool.ts` implementing the `Tool` interface from `src/types.ts`.
2. Export it from `src/tools/index.ts` in the `tools` record.
3. Add `<button id="your-tool" class="control tool" data-category="<category>">` to `#top-tools` in `index.html`. The `data-category` value must match one of the categories in `#category-col` (pen, brush, toy, stamp, eraser, filter, bucket, letter), or add a new category button and a matching `#top-tools[data-active-category="..."] .tool[data-category="..."]` rule in `style.css`.
4. `addListeners()` picks it up automatically via event delegation on `.tool`.
5. Run `npm run check` to verify types and tests pass.
6. Run `npm run build` to regenerate `script.js`.
